# Chapter 5: Research & Knowledge Synthesis

You need to make an informed decision â€” choosing a technology, evaluating a competitor, or understanding a market trend. Today, that means opening 20 browser tabs, skimming articles, losing track of which source said what, and ultimately making a decision based on incomplete information and gut feeling.

**The problem isn't finding information. It's synthesizing it.**

This chapter teaches you to delegate research workflows to your General Agent. Instead of manually collecting, reading, and comparing sources, you'll describe what you need to know, let the agent gather and structure the information, then apply your judgment to a well-organized synthesis.

## Principles Applied

| Principle | How It Applies |
|-----------|---------------|
| **Code as Universal Interface** | Research queries expressed as structured prompts, not vague questions |
| **Verification as Core Step** | Cross-reference claims across sources; never trust a single result |
| **Persisting State in Files** | Save research artifacts as markdown files for future reference |
| **Observability** | Agent shows its sources, reasoning, and confidence levels |

## Interface Focus

**Primary**: Cowork (research is conversational and iterative)
**Secondary**: Code (for structured output formatting)

## What You'll Learn

By the end of this chapter, you'll be able to:

- Frame research questions that produce actionable results
- Use your General Agent to gather, compare, and synthesize information from multiple sources
- Create structured comparison documents (feature matrices, pros/cons, decision frameworks)
- Verify claims and cross-reference sources systematically
- Build a personal knowledge base from research outputs
